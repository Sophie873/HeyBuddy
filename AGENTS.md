# 프로젝트 실행 지침

## 목표
- 마이크 음성 입력 -> 텍스트 변환 -> CLI 에이전트 실행 -> 음성 출력의 실시간 대화 흐름을 만들고, 가능한 한 지연 없이 명령을 처리한다.

## 기본 규칙
- 기본 언어: 한국어.
- 사용자 상호작용은 음성 기반 `python app.py` 우선이며, 텍스트 모드는 `python app.py --text`로만 동작한다.
- 실패 대비:
  - STT 실패 시 재시도 가능하도록 다시 듣기로 자연스럽게 복귀.
  - CLI 실행 오류는 화면에 에러를 남기고 계속 루프.
- 음성 출력은 기본 활성화 (`VOICE_TTS_ENABLED=true`).

## 코드 편집 원칙
- 기능 변경은 `app.py`에서 우선 진행하고, 동작 정책은 `.env` 환경변수로 제어한다.
- 에이전트 호출 실패 시 하드크래시 대신 사용자 메시지 출력으로 복구한다.
- 불필요한 임시 파일/로그 파일을 누적하지 않는다.

## 우선순위
1. 기본 동작 안정성
2. STT/TTS 전환성(온라인/오프라인)
3. 명령 중단·취소 경험 개선
4. 문서와 사용성 정리
